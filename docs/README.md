# README.md - Project Context (Screeps)

This file provides context and instructions for this **Screeps** game automation repository.

## 🚀 Project Overview

Automation script for the strategy MMO game **Screeps**. The code is focused on scalability, resilience, and logistical efficiency.

### Technologies and Architecture

- **Language:** TypeScript (compiled to ES2018).
- **Build System:** Webpack for minification and bundling into a single `main.js`.
- **Structure:** Modular based on Roles in `.ts` files.
- **Spawn Management:** Smart spawner with pre-spawning of creeps. Combat units (Guards/Archers) are only spawned with 15+ extensions. Global limit of 1 Scout with minimum priority to avoid economic deadlocks.
- **Logistics:** Static mining (Harvesters), dynamic logistics (Suppliers), and dedicated storage for upgrades (Controller Container).

## 📁 Project File Structure (Code)

- `src/main.ts`: Main loop and global orchestration.
- `src/structures/structure*.ts`: Structure planning intelligence, with verification and reconstruction, ignoring unsafe areas.
- `src/rooms/room*.ts`: Room planning intelligence.
- `src/spawns/spawn*.ts`: Creep generation intelligence.
- `src/tools/tool*.ts`: General tools.
- `src/roles/role*.ts`: Specific creep behaviors with target persistence in memory to avoid oscillations.
- `src/tasks/task*.ts`: Reusable task modules (Build, Upgrade, CollectEnergy, Repair).
- `dist/main.js`: Final file generated by Webpack for deployment.

## 🛠️ Commands and Operations

- **Change Workflow:** Whenever you make a change, you must first perform a `git commit` (with a descriptive message) and a `git push`. Only after that, run the `npm run deploy` command for deployment.
- **Automatic Deployment:** Run `npm run deploy`. This cleans `dist/`, compiles the code, increments the deploy counter, sends it to GitHub, and performs the upload via `screeps-api` to the private server.
- **Deploy Configuration:** Private server credentials are kept in the `screeps.json` file (based on `screeps.json.example`).

## 📝 Development Conventions (Surgical Changes)

- **Modularity & No Repetition:** Whenever possible, organize the logic into separate modules (e.g., `role.*.ts`, `task.*.ts`, `manager.*.ts`) to avoid code repetition and promote maintainability.
- **TypeScript Strict:** Maintain rigorous typing to avoid runtime errors.
- **ES Modules:** Use `import` and `export default` in the `src/` files.
- **CPU Efficiency:** Use `reusePath` in movements and avoid excessive `room.find` inside the loop.
- **Memory Safety:** Always check if objects exist in `Game.getObjectById` before using them.
- **Screeps Compatibility:** TS target should be `ES2018` for compatibility with the game environment.
